在vue 3中的css module中实现深度选择

个人感觉 CSS Module 更适合需要复用 css 的情况，对 css 复用要求不高的话普通的 Scoped CSS 就够了。两者之间实际使用起来感觉没什么太大区别。

CSS Module + :global

css module 提供了一个 :global 选择器来实现类似深度选择器的功能，这个特殊的选择器可以在全局范围内选中元素，具体就是被选中的 class 在编译时不会加上 hash 。

下例中前缀的 .a 没有 :global 选择器，因此 .a 还是会被 hash ，这就保证了样式的影响范围依然在组件内。

如果使用 scss 或者 less 等 css 预处理器的话，可以使用 :global { ... } 将多个子选择器组织在全局空间中。

其他各种组合的编译结果

CSS Module without :global

不加 :global 的话前后两个 class 都会加上 hash ，由于内部的子组件中的 class 没有 hash ，因此无法生效。

CSS Module + 第一级就是:global

使用 :global(...) 时，无论是 css 还是 less 都无法传递全局的作用范围，使用 :global { ... } 则可以将全局作用范围传递下去。

Scoped CSS + :global

scoped css 也有 :global 选择器，但是这样无法将作用范围限制在组件内，因为会丢失前缀的 class 。

Scoped CSS + :deep

Scoped CSS + 第一级就是:deep

即使在第一级就使用 :deep ，依然能保证作用域在组件的范围内，只不过缺少了 class 的话性能会降低。
