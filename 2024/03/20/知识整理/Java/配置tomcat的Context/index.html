有时需要部署的文件不在 webapps 目录下，有时可能需要将 app 的 context path 设置为多级目录，这些可以通过配置 <Context /> 来实现。

Tomcat 官方建议在 $CATALINA_BASE/conf/[enginename]/[hostname]/ 目录下添加各应用的 Context 设置，通常情况下这个目录的路径是 $CATALINA_BASE/conf/Catalina/localhost/ ，目录下的每一个 xml 文件都对应一个 app 的 Context 设置。

文件名会被解析为 Context Path ，文件名中的 # 号会被替换成 / 。例如：

  • manager.xml 的 Context Path 为 /manager
  • hello#world.xml 的 Context Path 为 /hello/world
