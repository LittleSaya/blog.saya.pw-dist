统一依赖管理，以及统一依赖和插件的版本管理

在父模块 pom 文件的 /project/dependencyManagement/dependencies 中声明的依赖版本可以直接被子模块使用，子模块在声明相同的依赖时，可以忽略 <version> 标签。

在父模块 pom 文件的 /project/build/pluginManagement/plugins 中定义的插件版本和配置可以直接被子模块使用，子模块在使用相同的插件时，可以忽略插件的版本和配置。

在父模块 pom 文件的 /project/dependencies 中声明的依赖会传递到子模块，子模块不需要再声明相同的依赖。

插件的默认 groupId

如果在使用插件时没有指定 <groupId> ，则默认为 org.apache.maven.plugins 。
