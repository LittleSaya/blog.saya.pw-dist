前言

去年的时候整了一个 haproxy + v2ray + nginx 的，但是因为图省事没有买域名，也没有申请证书，所以用一阵子就被屏蔽了。今天吸取了一下经验教训，重新搭建了一个，这次不图省事了，买了域名也申请了证书。因为还是有被屏蔽的可能，所以在此记录一下搭建过程，如果下一次还要搭建，就不用在网上找来找去了。另外如果你是路人，碰巧你也有科学上网的需求的话，也可以参考一下我这篇文章的内容，希望能够对你有所帮助。

服务器

购买服务器的选择很多，有 bandwagonhost 、 digital-vm 和 sugarhosts 等

bandwagonhost 在国内可以通过 https://bwh81.net 访问（直接点击会带营销编号访问）

digital-vm 可以通过 https://digital-vm.com 访问（直接点击会带营销编号访问）

sugarhosts 可以通过 https://www.sugarhosts.com 访问

其中 digital-vm 的访问速度稍微有点慢，日本服务器会绕道香港和美国西海岸，不是很推荐。

服务器的购买和登录就不细说了，这些太基本，如果你对此有疑问的话，可以询问商家，并且自学一下 ssh 客户端的使用方法。

域名

域名可以在 https://www.namesilo.com 购买。 top 域名比较便宜，第一年大约 13 元人民币，续订价格大约 35 元人民币（撰文时汇率）。支持 paypal 和支付宝。

买完域名以后要设置域名的 DNS 记录，先点击首页上的 My Account ，然后在 Dashboard 里点击 Active Domains 卡片下的 Manage ， 然后在表格中点击刚刚购买的域名的名称，进入 Domain Console ，再点击 DNS Record 旁边的 Update ，进去以后增加或者修改一个 A 记录，把 IPV4 ADDRESS 设置为服务器的 IP 地址即可。

域名生效可能需要十几分钟的时间，可以在命令提示符下运行 nslookup <your.domain.name> 来检查域名是否能够被正确地解析。

V2ray 服务端

这一段介绍如何设置服务器，撰文时服务器的操作系统为 CentOS 7 ，我使用 putty 作为 ssh 客户端。

申请证书

我使用 Let’s Encrypt 申请证书，步骤如下：

yum install epel-release

yum install snapd

systemctl enable --now snapd.socket

ln -s /var/lib/snapd/snap /snap

reboot

snap install --classic certbot

ln -s /snap/bin/certbot /usr/bin/certbot

certbot certonly --standalone

仔细阅读，按要求填写即可。

生成的证书在 /etc/letsencrypt/live/<your.domain.name> 目录下，证书文件为 fullchain.pem 、私钥文件为 privkey.pem 。

v2ray 客户端

windows

Windows 有图形客户端，但是我不需要图形客户端那种很多服务器一起管理的功能，毕竟自己购买服务器的话，往往只有一台。所以，这里就简单点，直接用 v2ray 自己的客户端，然后写两个脚本配置一下 windows 的系统代理服务就好了。

从地址 https://github.com/v2fly/v2ray-core/releases/download/v5.12.1/v2ray-windows-64.zip 下载客户端，如果没法直接下载的话，可以先在服务器上下载好，然后使用随 putty 附带的 psftp.exe 或者 pscp.exe 连接服务器，再从服务器下载到本地。

假设把压缩文件解压到 C:\software\v2ray\ 目录下。

创建配置文件 C:\software\v2ray\config.json ：

展开 config.json 文件的内容

配置文件很长，但是结构还是比较清晰的，基本分成四部分：

  • log - 日志设置
  • inbounds - 入口，这里设置了一个 socks 端口和一个 http 端口
  • outbounds - 出口，这里设置了三个出口：
    • proxy - 走代理
    • direct - 不走代理
    • block - 阻止连接
  • routing - 路由，主要用于广告拦截和国内直连，最后有两个我自己加的 github 和 icons8 的配置

有几点需要特别注意：

  1. 将 <your.domain.name> 替换为你的域名
  2. 将 <uuid> 替换为之前设置服务端时生成的 uuid
  3. 将 wsSettings.path 设置为 /binary-path ，这个值必须和 nginx 配置中的 location 后的值，以及和服务端 v2ray 配置中的 wsSettings.path 严格相等
